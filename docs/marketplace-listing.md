# Marketplace listing template

One reusable template for every place Candela is listed (VS Code Marketplace,
JetBrains, Open VSX, Zed, Sublime, Neovim, GitHub release, etc.). Keep the copy
here, not per-tool: the pitch is identical everywhere, only the install steps change.

**Rules**

- No theme counts anywhere ("14 light, 2 dark"). Adding a theme must never mean
  editing marketing copy. Say "a family", "light schemes and dark companions".
- Same tagline and same "Why Candela" block in every listing.
- Same screenshots in every listing, referenced by raw GitHub URL (below).

---

## Short description (tagline)

The one-liner shown in search results and under the title. Keep the first ~120
characters carrying the meaning, since some marketplaces truncate.

> Color themes tuned for long, comfortable coding sessions — low-glare
> backgrounds, desaturated accents, and WCAG-AA contrast throughout, with the
> same palette across your editor and terminal.

---

## Why Candela

- **Built for tired eyes** — low-glare backgrounds (never pure white or black) and
  desaturated accents that resist visual fringing.
- **Readable by design** — every syntax and UI color clears WCAG-AA contrast;
  primary text clears AAA.
- **One palette everywhere** — the same colors in your editor and your terminal.
- **A whole family** — light schemes for every room, plus dark companions, all
  from one coordinated, curated set.

## Screenshots

Committed images live in `docs/screenshots/candela-<id>.png`, one per theme,
regenerated by `cd app && npm run screenshots`. Reference them by **raw GitHub
URL** so every marketplace renders the same set:

```
https://raw.githubusercontent.com/schovi/candela-themes/main/docs/screenshots/candela-<id>.png
```

**Suggested picks** (variety over favorites — show the range):

| Slot | Theme id | Why |
| --- | --- | --- |
| Hero | `sepia-paper` | warm, the signature look |
| Light A | `slate-mist` | cool / neutral gray |
| Light B | `meadow` or `lagoon` | a colored light scheme |
| Dark | `nocturne` or `borealis` | the two dark companions |

The two dark themes are `nocturne` and `borealis`; everything else is light.

## Per-marketplace notes

- **VS Code / Open VSX** — description is the bundled `README.md`
  (`lib/emitters.js` → `vscodeReadme()`); tagline is `package.json` `description`.
  Markdown + remote images render.
- **JetBrains** — `plugin.xml` `<description>` (CDATA, HTML not Markdown); short
  tagline reused from `DESCRIPTION`.
- **Sublime / Neovim** — plain README in the archive; keep install block, drop
  screenshots if the host doesn't render remote images.
- **Zed / GitHub** — full Markdown renders; use the template verbatim.
